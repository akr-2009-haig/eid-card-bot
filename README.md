🎉 تصميم بوت بطاقات تهنئة عيد الفطر

## التشغيل السريع

1. حدّث ملف `config.py` بالقيم الحقيقية التالية:
   - `BOT_TOKEN`
   - `API_ID`
   - `API_HASH`
2. ثبّت الاعتمادات:

```bash
pip install -r requirements.txt
```

3. شغّل البوت:

```bash
python main.py
```

إذا كانت الإعدادات ناقصة سيطبع البوت رسالة واضحة بدل الخطأ الداخلي القادم من Pyrogram.

## وثيقة الاستضافة

- راجع الملف `HOSTING.md` لمعرفة الاستضافات المجانية والرخيصة المناسبة للبوت.

## تفاصيل تنفيذ لوحة الأدمن

### 🎨 إدارة القوالب

**ماذا يرى الأدمن**

- يدخل `/admin`
- يضغط `🎨 إدارة القوالب`
- يرسل البوت:

```text
🎨 إدارة قوالب بطاقات العيد
يمكنك من هنا إدارة القوالب التي يستخدمها البوت لتصميم بطاقات التهنئة.
```

**ماذا يضغط**

- `➕ إضافة قالب`
- `🗑 حذف قالب`
- `📂 عرض القوالب`
- `⬅️ رجوع`

#### ➕ إضافة قالب

**رد البوت**

```text
📤 أرسل الصورة التي تريد إضافتها كقالب.
يمكنك إرسال PNG أو JPG أو JPEG، وسيتم حفظها داخل مجلد القوالب واستخدامها تلقائيًا عند التصميم.
```

**ماذا يحدث في النظام**

1. تحميل الصورة من تيليجرام.
2. إنشاء اسم عشوائي للملف داخل `data/templates/`.
3. تشغيل OCR - عند توفر `pytesseract` - للبحث عن:
   - `الاسم`
   - `[الاسم]`
   - `(الاسم)`
   - `اسم`
4. حفظ مسار القالب وبيانات موضع الاسم في قاعدة البيانات.

**رد النجاح**

```text
✅ تم إضافة القالب بنجاح
🖼 أصبح القالب متاح الآن لاستخدامه في تصميم البطاقات.
```

إذا تم اكتشاف موضع الاسم يرسل البوت إحداثياته، وإذا لم يتم اكتشافه يستخدم الموضع الافتراضي الحالي.

#### 📂 عرض القوالب

**رد البوت**

```text
📂 القوالب الحالية في البوت
عدد القوالب: X
```

ثم يرسل كل قالب كصورة أو ملف مع:

- رقم القالب
- اسم الملف
- تاريخ الإضافة

#### 🗑 حذف قالب

**رد البوت**

```text
🗑 اختر القالب الذي تريد حذفه
```

ثم تظهر الأزرار:

- `قالب 1`
- `قالب 2`
- `قالب 3`

**ماذا يحدث في النظام**

1. حذف السجل من جدول القوالب.
2. حذف الملف من `data/templates/`.

**رد البوت**

```text
✅ تم حذف القالب بنجاح
```

#### 🎨 استخدام القوالب وقت التصميم

عند إرسال المستخدم لاسمه:

1. يقرأ البوت جميع القوالب.
2. يختار قالبًا عشوائيًا.
3. إذا كان القالب يحتوي إحداثيات اسم مستخرجة بالـ OCR:
   - يغطي مكان النص القديم
   - يكتب اسم المستخدم فوقه
4. إذا لم توجد إحداثيات:
   - يستخدم الموضع الافتراضي الحالي في منتصف التصميم
5. يحفظ البطاقة داخل `data/generated/`
6. يرسل الصورة للمستخدم

### 🔒 الاشتراك الإجباري

**ماذا يرى الأدمن**

- من `/admin`
- يضغط `🔒 الاشتراك الإجباري`
- يرسل البوت:

```text
🔒 إدارة الاشتراك الإجباري
يمكنك إضافة أو حذف القنوات أو القروبات التي يجب على المستخدم الاشتراك فيها قبل استخدام البوت.
```

**الأزرار**

- `➕ إضافة قناة أو قروب`
- `🗑 حذف قناة أو قروب`
- `📂 عرض القنوات والقروبات`
- `✏️ تعديل رسالة الاشتراك`
- `⬅️ رجوع`

#### ➕ إضافة قناة أو قروب

**رد البوت**

```text
📎 أرسل رابط القناة أو القروب
```

ويدعم:

- `@channelname`
- `https://t.me/channelname`
- `https://t.me/+AbCdEfGh`
- `https://t.me/joinchat/AbCdEfGh`

**ماذا يحدث في النظام**

1. تطبيع الرابط واستخراج قيمة البحث المناسبة.
2. محاولة قراءة بيانات القناة أو القروب من تيليجرام.
3. حفظ:
   - الرابط المعروض للمستخدم
   - اسم أو يوزر الجهة
   - `chat_id` لاستخدامه في التحقق من الاشتراك

**رد النجاح**

```text
✅ تم إضافة القناة أو القروب للاشتراك الإجباري
```

#### ماذا يرى المستخدم عند عدم الاشتراك

يرسل البوت رسالة الاشتراك المحفوظة في لوحة النصوص، ثم أزرار لكل قناة أو قروب، ثم:

- `✅ تحقق من الاشتراك`

وعند الضغط:

- إذا كان مشتركًا في الجميع: ينتقل للواجهة الرئيسية.
- إذا لم يكن مشتركًا: يظهر تنبيه:

```text
❌ يجب الاشتراك في جميع القنوات أولاً
```

#### 📂 عرض القنوات والقروبات

يعرض البوت قائمة مرقمة، وكل عنصر يحتوي:

- النوع: قناة أو قروب
- الاسم
- الرابط

#### 🗑 حذف قناة أو قروب

يعرض البوت الجهات كأزرار للحذف، وبعد الاختيار يرسل:

```text
✅ تم حذف القناة من الاشتراك الإجباري
```
