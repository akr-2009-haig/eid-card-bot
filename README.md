🎉 تصميم بوت بطاقات تهنئة عيد الفطر

## التشغيل السريع

1. اضبط متغيرات البيئة التالية:
   - `BOT_TOKEN`
   - `API_ID`
   - `API_HASH`
   - `ADMIN_IDS` (اختياري، قائمة مفصولة بفواصل)
2. ثبّت الاعتمادات:

```bash
pip install -r requirements.txt
```

3. شغّل البوت:

```bash
python main.py
```

إذا كانت الإعدادات ناقصة سيطبع البوت رسالة واضحة بدل الخطأ الداخلي القادم من Pyrogram.

يمكنك أيضًا ترك القيم الافتراضية داخل `config.py` أثناء التطوير المحلي، لكن في Render يُفضّل استخدام متغيرات البيئة فقط.
يُستخدم `STORAGE_DIR` للبيانات المتغيرة فقط مثل قاعدة البيانات والقوالب والبطاقات الناتجة وملفات السجل، أما الخط الافتراضي `data/fonts/arabic.ttf` فيبقى مرفقًا مع المشروع.

## وثيقة الاستضافة

- راجع الملف `HOSTING.md` لمعرفة الاستضافات المجانية والرخيصة المناسبة للبوت.
- يوجد ملف `render.yaml` جاهز للنشر على Render كخدمة `worker` مع `STORAGE_DIR=/var/data` لاستخدام قرص دائم، وتم تثبيت إصدار Python في Render على `3.11.8` عبر `.python-version` و `PYTHON_VERSION`.

## تفاصيل تنفيذ لوحة الأدمن

### 🎨 إدارة القوالب

هذا القسم يسمح للأدمن بإدارة قوالب بطاقات التهنئة التي يستخدمها البوت عند تصميم الصور.  
القوالب عبارة عن صور ثابتة يرفعها الأدمن، ثم يستخدمها البوت لاحقًا عند إنشاء البطاقة للمستخدم.

**خصائص القسم**

- يدعم جميع مقاسات الصور
- يدعم عددًا غير محدود من القوالب
- يختار قالبًا عشوائيًا عند التصميم
- يحفظ القوالب داخل `data/templates/`
- يحفظ البطاقات النهائية داخل `data/generated/`

#### 👑 الوصول إلى لوحة القوالب

**ماذا يرى الأدمن**

1. يدخل إلى `/admin`
2. يضغط `🎨 إدارة القوالب`

**ماذا يرد البوت**

```text
🎨 إدارة قوالب بطاقات العيد
يمكنك من هنا إدارة القوالب التي يستخدمها البوت لتصميم بطاقات التهنئة.

• يدعم جميع مقاسات الصور
• يمكن إضافة عدد غير محدود من القوالب
• يتم اختيار قالب عشوائي عند تصميم البطاقة
• يتم حفظ القوالب داخل data/templates/
```

**ما الأزرار التي تظهر**

- `➕ إضافة قالب`
- `🗑 حذف قالب`
- `📂 عرض القوالب`
- `⬅️ رجوع`

---

#### ➕ إضافة قالب

**ماذا يضغط الأدمن**

- يضغط `➕ إضافة قالب`

**ماذا يرد البوت**

```text
📤 أرسل الصورة التي تريد إضافتها كقالب.
يمكنك إرسال أي صورة بأي مقاس بصيغة PNG أو JPG أو JPEG.
سيتم حفظها داخل data/templates/ ثم تشغيل OCR للبحث عن كلمة الاسم أو [الاسم] أو (الاسم) لاستخدامها تلقائيًا عند التصميم.
```

**ماذا يحدث في النظام**

1. تحميل الصورة من تيليجرام.
2. إنشاء اسم عشوائي للملف داخل `data/templates/`.
3. حفظ نسخة القالب داخل المجلد.
4. تشغيل OCR - عند توفر `pytesseract` - لقراءة النصوص داخل الصورة.
5. البحث عن أي صيغة لاسم المكان المخصص مثل:
   - `الاسم`
   - `[الاسم]`
   - `(الاسم)`
   - `اسم`
6. عند العثور على النص:
   - يتم أخذ الإحداثيات `X` و`Y`
   - يتم أخذ العرض `W` والارتفاع `H`
   - يتم تقدير `font_size`
7. يتم حفظ مسار الملف وبيانات الـ OCR في قاعدة البيانات.

**ماذا يرد البوت بعد النجاح**

```text
✅ تم إضافة القالب بنجاح
🖼 أصبح القالب متاح الآن لاستخدامه في تصميم البطاقات.

📁 رقم القالب: 7
🗂 تم حفظ الملف داخل: data/templates/template_xxxxx.png
🔎 OCR: تم اكتشاف موضع الاسم تلقائيًا
• النص المكتشف: [الاسم]
• X=520 | Y=760
• W=120 | H=40
• حجم الخط المقترح: 60
```

**إذا لم ينجح OCR**

يرد البوت برسالة نجاح أيضًا، لكن بدل تفاصيل الإحداثيات يرسل:

```text
ℹ️ OCR: لم يتم العثور على كلمة الاسم تلقائيًا، وسيستخدم البوت الموضع الافتراضي الحالي.
```

#### 🎨 استخدام القوالب عند التصميم

عندما يرسل المستخدم اسمه، يحدث التالي:

1. يقرأ البوت جميع القوالب من قاعدة البيانات.
2. يختار قالبًا عشوائيًا.
3. يفتح صورة القالب.
4. إذا كان القالب يحتوي على بيانات OCR:
   - يغطي مكان النص القديم
   - يكتب اسم المستخدم في نفس الموضع
   - يضبط حجم الخط بحسب المساحة المتاحة
5. إذا لم توجد بيانات OCR:
   - يستخدم الموضع الافتراضي الحالي
   - يكتب `تهنئة من` ثم الاسم في منتصف التصميم
6. يحفظ البطاقة الناتجة داخل `data/generated/`.
7. يرسل الصورة للمستخدم.

**مثال عملي**

- القالب يحتوي:

```text
🌙 عيد فطر مبارك
كل عام وأنتم بخير
تهنئة من [الاسم]
```

- المستخدم يرسل:

```text
أحمد
```

- النتيجة التي ينشئها البوت:

```text
🌙 عيد فطر مبارك
كل عام وأنتم بخير
تهنئة من أحمد
```
