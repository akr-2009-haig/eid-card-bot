# خيارات استضافة البوت

هذه الخيارات مناسبة لبوت تيليجرام Python/Pyrogram مثل هذا المشروع:

## 1) Render
- **النوع:** سحابي
- **الخطة المجانية:** متوفرة لكن الخدمة قد تنام عند عدم الاستخدام
- **الخطة الرخيصة:** تبدأ عادةً من باقة شهرية منخفضة
- **مناسب لـ:** التجارب والبوتات الصغيرة إلى المتوسطة
- **الميزات:** نشر سهل من GitHub، متغيرات بيئة، سجلات مباشرة
- **الإعداد الحالي في هذا المشروع:** يوجد ملف `render.yaml` جاهز للنشر كبوت polling عبر `worker` وليس `web service`، وتم تثبيت Python على `3.11.8` عبر `.python-version` و `PYTHON_VERSION`
- **التخزين:** تم ضبط المشروع ليدعم `STORAGE_DIR=/var/data` حتى تحفظ قاعدة SQLite والقوالب والملفات الناتجة والسجلات على القرص الدائم في Render

## 2) Railway
- **النوع:** سحابي
- **الخطة المجانية:** قد تكون محدودة حسب الرصيد أو العروض الحالية
- **الخطة الرخيصة:** مناسبة للبوتات الصغيرة
- **مناسب لـ:** من يريد إعدادًا سريعًا وواجهة سهلة
- **الميزات:** نشر مباشر، دعم متغيرات البيئة، إدارة خدمات بسيطة

## 3) Koyeb
- **النوع:** سحابي
- **الخطة المجانية:** متوفرة عادةً بموارد محدودة
- **الخطة الرخيصة:** جيدة لتشغيل دائم بتكلفة منخفضة
- **مناسب لـ:** البوتات التي تحتاج استقرارًا أفضل من الخطط المجانية المؤقتة

## 4) VPS رخيص
- **أمثلة:** Contabo / Hostinger VPS / Hetzner
- **التكلفة:** منخفضة شهريًا مقارنة بالاستضافات المدارة
- **مناسب لـ:** التشغيل الدائم والتحكم الكامل
- **الميزات:** أفضل خيار عملي إذا كان البوت سيعمل 24/7 وبعدد مستخدمين متزايد

## التوصية
- **مجاني للتجربة:** Render أو Koyeb
- **رخيص وعملي للإنتاج:** VPS رخيص أو Railway حسب الميزانية وسهولة الإدارة

## ملاحظات مهمة قبل النشر
- في Render ضع القيم الحقيقية في متغيرات البيئة التالية:
  - `BOT_TOKEN`
  - `API_ID`
  - `API_HASH`
  - `ADMIN_IDS`
- تأكد من تثبيت الاعتمادات عبر:

```bash
pip install -r requirements.txt
```

- ثم شغّل البوت:

```bash
python main.py
```

## خطوات النشر على Render
1. اربط المستودع بـ Render.
2. اختر **Blueprint** أو ارفع الملف `render.yaml`.
3. أنشئ القيم السرية من لوحة Render لـ:
   - `BOT_TOKEN`
   - `API_ID`
   - `API_HASH`
   - `ADMIN_IDS`
4. اترك `STORAGE_DIR` على `/var/data` كما هو في `render.yaml`.
5. بعد أول نشر سيعمل البوت كخدمة خلفية (`worker`) ويحتفظ ببيانات SQLite والملفات القابلة للتغيير على القرص الدائم.
